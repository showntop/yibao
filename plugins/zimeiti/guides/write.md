# 成文框架（write）

把选题写成一篇能发的稿子。原则：先完成，再完美——初稿追求结构完整，不逐句打磨。

## 结构（默认五段式）

1. **钩子**：开头 3 行内给出「为什么要读下去」——一个具体场景、一个反常识判断、或一个尖锐问题。不要从定义和背景开始。
2. **痛点共鸣**：把选题的痛点写成读者正在经历的事，用「你」而不是「大家」。
3. **核心内容**：3 个以内的要点，每个要点 = 观点 + 一个具体例子。要点之间有过渡句，不要硬跳。
4. **转折/深化**：把视角抬一层——从「怎么办」到「为什么是这样」，或给一个反例让论述立得住。
5. **收尾**：一个可带走的结论或行动，配一句有余味的话。不要总结全文。

## 语气

- 口语优先：能说的词不写的词，短句优先，一段不超过 4 行。
- 克制：不堆形容词，不用「非常」「极其」，惊叹号全文不超过 2 个。
- 具体：数字、名字、场景优于抽象判断（「3 天涨粉 200」优于「效果不错」）。

## 流程

1. 先列 3-5 句的骨架（每段一句），确认方向再展开。
2. 初稿一气呵成写完，不回头改。
3. 改稿只动被点名的部分，不顺手重写写好的段落。

## 平台适配（有 platform 时）

- 公众号：标题带悬念，段落短，配图位留标注。
- 小红书：emoji 分段，清单体，首图文案即标题。
- 播客：口播稿，全短句，去掉一切书面语和括号注释。
