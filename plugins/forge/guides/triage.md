# 快筛方法论（2 分钟判生死）

> 目的：大多数想法应该死在这一环。快筛要便宜、要狠、要有结论。
> 快筛不是深分析——它是「值不值得花 15 分钟挑战」的判断。

## 流程

1. **先召回记忆**：调 `forge.guide` 之外不需要工具——直接想（或问）：这个想法和用户历史裁决过的方向像不像？如果记忆里有相关否决，**必须摆出来**：「你之前否过类似的 X，理由是 Y——是情况变了，还是想翻案？」
2. **问 2-3 个关键问题**（不够才问，不要全问）：
   - 谁有？——具体到什么角色、什么场景下的人
   - 多痛、多频？——是「不得不解决」还是「有更好」？多久遇到一次？
   - 现在怎么解决的？——没有现有方案的痛点多半不是痛点（是没人有这个场景）
   - （可选）凭什么是你做？——用户的独特资源/经历/触达
3. **出快筛卡**，格式固定：

```
快筛卡
- 痛点真实性：真 / 伪 / 存疑（一句理由）
- 现有方案：（用户现在怎么解决的，含竞品/土办法）
- 与你的匹配度：高 / 中 / 低（技能、资源、触达）
- 历史比对：（记忆里有无类似裁决，有就引用理由）
- 建议：进挑战 / 直接搁置（一句理由）
```

4. **落库**：调 `forge.triage` 存快筛卡（triage 字段），status 取：
   - `快筛过`——值得花 15 分钟挑战
   - `已搁置`——现在不值得做（**这不需要用户确认，是快筛的本职判断**，但要在对话里说清理由）

## 纪律

- **不超 3 个问题**。问多了就是挑战，不是快筛。
- **必须有结论**。不许「这个方向很有意思，值得探索」——要么进挑战，要么搁置，给理由。
- **伪痛点直接说**。用户问「是不是痛点」时，答「不是」也是价值。
