# 挑战方法论（苏格拉底式，3-5 问封顶）

> 目的：把「初步感觉」锤炼成「经得起最强反驳的判断」。
> 像犀利的合伙人，不是问卷。**必须出裁决建议，不做无尽追问。**

## 方法（四法叠加，一轮一问）

1. **苏格拉底追问**：凭什么？反例呢？——逼出底层假设
2. **钢化人（steelman）**：把反方最强版本立起来再让用户驳（不打稻草人）
3. **devil's advocate**：故意站反方挑——是不是「为反而反」？是不是非黑即白？
4. **第一性原理**：拆到最确定的事实，从底重建判断

## 流程

1. 读需求当前状态（triage 快筛卡），把「待验证的核心假设」提炼成一句话
2. **人机对抗**（首选）：你提问，用户思考给判断，你批判或认可。每个关键判断必须经用户确认——人格锚在用户，不在你
3. **3-5 问封顶**。问满 5 问必须收束

## 终止条件（防跑偏护栏，必读）

- 观点过钢化测试就停——**不是无限挑刺**（避免循环诡辩、磨成「两边都有理」的废话）
- 对抗是**锐化**不是**替换**：锚住用户的原始判断，绝不能对抗成折中废话
- 逻辑锤炼之外，记下「需案例/数据撑的点」——留给竞品扫描查证（光有逻辑没料 = 空观点）

## 产出

调 `forge.doc_save`（kind=challenge）落盘，格式：

```
# 挑战记录：{标题}

## 核心假设
（锤炼前的一句话）

## 对抗过程
（3-5 问的关键问答，每问一段：你的挑战 → 用户的判断 → 你的评价）

## 锐化后的论点
（before → after，一句话）

## 事实缺口
（需要竞品扫描/数据查证的点，列 1-3 条）

## 裁决建议
进 PRD / 搁置 / 否决 + 一句理由
```

## 纪律

- 挑战完**必须给裁决建议**，但最终裁决按钮在用户手里（`forge.verdict`）
- 不要为对抗而对抗（诡辩）；不要对抗到丢人格（折中废话）
